社交分享的实现

在iOS中，实现社交分享的方法

  • 自己编写各个平台的分享代码（代码量较多）
  • 利用iOS自带的Social.framework
  • 利用第三方的分享框架
    • 友盟分享：http://dev.umeng.com/social/ios/share/quick-integration
    • ShareSDK：http://wiki.mob.com/iOS快速集成指南
    • 百度社会化分享组件：http://developer.baidu.com/soc/share
      （百度还有个“社会化登录组件”：http://developer.baidu.com/soc/login）

分享的授权机制

  • OAuth
  • SSO

OAuth

OAuth2.0
概念: 互联网标准协议,用来对资源进行开放授权(Open Authorization)
核心: 应用不需要获取平台的账号密码即可使用平台功能
优势: 提高了平台账号的安全性

目前几乎所有主流平台都使用OAuth2.0作为主要授权方式

OAuth的实现

移动端OAuth实现:
使用UIWebView访问url请求auth code
相当于客户端没有直接参与授权的过程

弊端:
首次授权会以网页的形式让用户输入平台登录信息,使操作繁琐
即使是以网页的形式,仍然处在开发者的应用内,安全性较低(开发者的应用制作假授权界面,用户无法辨别)

SSO

概念: Single Sign On，单点登录技术，在多个应用系统中，用户只需要登录一次就可以访问所有相互信任的应用系统

移动端效果: 安装并登录微博App,且开发者应用集成SSO授权,首次授权会直接进入微博App确认授权,过程中不需要输入账号密码

优势: 简化了授权过程,减少输入操作(免输入登录信息),更加安全

是OAuth授权机制的优化方案
