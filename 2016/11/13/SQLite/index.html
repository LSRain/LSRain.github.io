系统提供的数据存储方式的弊端

  • 不方便操作大量的数据
    • 如果要添加新的数据,必须先把旧数据全部加载到内存中.
    • 系统提供的数据存储方式都是覆盖存储的,新的数据会覆盖旧的数据.
  • 不方便查找大量的数据
    • 当数据量非常庞大时,要查询其中某些数据,就非常困难.
  • 比如把数组数据存储到plist文件中,当数据量比较大时,无论存储还是查找数据都相当不方便.
    • 当plist文件中的数据量非常庞大时,一次性加载到内存,会造成内存暴涨
    • 当plist文件中的数据量非常庞大时,要查询其中的某一条数据时非常困难

SQLite 简介

  • SQLite是一款轻量级数据库

    • 设计目标是嵌入式的(可以放在手机沙盒里面的)
    • 体积小,占用资源少
    • 处理速度快
    • 跨平台

  • 官方网站：http://www.sqlite.org/

  • 客户端的数据库 : SQLite

  • 服务器端的数据库 : MySQL , SQLServer , Oracle

小结

  • 新增 ／ 修改 ／ 删除本质上都是执行一个 SQL

  • 新增记录可以使用 sqlite3_last_insert_rowid 获取最后插入的主键数值

  • 更新 / 删除记录可以使用 sqlite3_changes 获取影响数据行数

  • 在开发数据库功能时，最容易出错的位置是 SQL 语句的错误，而不是代码错误，一定要利用 navicat 协助检查语法

事务

  • 事务在开发使用的并不多.但是,是资深程序猿必须知道的.

  • 使用场景 : 大规模数据操作时使用.

  • 使用步骤 :

使用步骤 步骤说明
1 在准备做大规模数据操作前，首先开启一个事务，保存操作前的数据库的状态[快照]
2 开始数据操作
3 如果数据操作成功，提交事务，让数据库更新到数据操作后的状态
4 如果数据操作失败，回滚事务，让数据库还原到操作前的状态

如果没有显式的开启事务，SQLite 会 :

  • 在每一次数据库操作前开启事务(非常影响数据库的性能)
  • 操作完成后，自动提交事务;
  • 性能差!!!

使用事务可以 :

  • 数据库操作之前,建立快照
  • 数据库操作完成之后,提交快照
  • 提高数据库操作性能
  • 保证数据完整性

事务 SQL

  • 开启事务 BEGIN TRANSACTION;
  • 提交事务 COMMIT TRANSACTION;
  • 回滚事务 ROLLBACK TRANSACTION;
